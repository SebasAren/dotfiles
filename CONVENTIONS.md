# Development Conventions

## Code Style

### Lua
- 2-space indentation
- Format with [StyLua](https://github.com/JohnnyMorganz/StyLua) (`mise run format-lua`)
- Lint with [luacheck](https://github.com/mpeterv/luacheck) (`mise run lint-lua`)
- Use `---@type` annotations for LuaLS
- Use `snake_case` for functions and variables
- Prefer `require().setup()` over `vim.cmd()` for plugin configuration

### Python
- 4-space indentation
- Lint with [ruff](https://github.com/astral-sh/ruff) (`mise run lint-python`)
- Format with [ruff](https://github.com/astral-sh/ruff) (`mise run format-python`)
- `snake_case` for functions and variables
- `PascalCase` for classes

### Shell
- Use `#!/usr/bin/env bash` and `set -euo pipefail`
- Lint with [shellcheck](https://github.com/koalaman/shellcheck) (`mise run lint-shell`)
- Format with [shfmt](https://github.com/mvdan/sh) (`shfmt -w .`)
- Lowercase filenames with hyphens for multi-word names
- One concern per file (see `.bashrc.d/` structure)

### TypeScript/JavaScript
- Typecheck with `mise run typecheck` (per-package `tsc --noEmit`)
- Format with `mise run format-ts` (prettier) or prettierd in Neovim
- Avoid `as any` and `@ts-ignore`
- Use `spawnSync` with array args (no shell escaping needed). `Bun.escapeShellArg` is undefined in Bun 1.3.x; `Bun.makeTempDir` does not exist — use `mkdtempSync` from `node:fs` instead.

## Git Workflow

- **Conventional commits**: `<type>(<scope>): <description>`
  - Types: `feat`, `fix`, `docs`, `style`, `refactor`, `perf`, `test`, `build`, `ci`, `chore`, `revert`
  - Example: `feat(nvim): add rust-analyzer LSP config`
- **Atomic commits**: one logical change per commit
- **Pre-commit checks**: run `mise run pre-commit` (format + lint + typecheck + tests). No git hook is auto-installed; invoke the task manually or via your worktree workflow.
- **Commit messages**: optionally generated by pi via worktrunk

## Stow Package Management

- Use GNU Stow for all configuration symlinks
- Each tool gets its own top-level directory
- Config files go in their standard XDG location within the tool directory: `tool-name/.config/tool/`
- Edit files in the repo, never in the symlink target
- Avoid hard-coded paths that assume a specific username or machine


## Neovim

- Plugin specs in `lua/plugins/` — each file returns a Lazy.nvim table
- LSP server configs in `lsp/*.lua` — one file per server
- Use `pcall(require, "custom-settings")` for machine-specific overrides (file is gitignored)
- Keybinding conventions: `<space>` as leader, `gd` for definition, `gr` for references
- Never use `vim.cmd("packadd")` — Lazy.nvim handles loading

## Secrets

- Never commit secrets or credentials to the repo
- Use `~/.secrets.tpl` with Proton Pass CLI template syntax: `{{ pass://path/to/key }}`
- Secrets are resolved lazily on first use, not at shell startup
- API keys needed: `EXA_API_KEY`, `CONTEXT7_API_KEY` (for pi extensions)

## File Organization

```
.
├── [tool]/.config/[tool]/     # XDG-compliant config location
├── .mise/tasks/               # mise task definitions (pre-commit, format, lint, etc.)
├── bashrc/.bashrc.d/          # Modular shell scripts
├── mise.toml                  # Tool versions
├── AGENTS.md                  # Agent-facing guide
├── CONVENTIONS.md             # This file
└── README.md                  # Human-facing setup guide
```

## Pre-commit Checks

Run `mise run pre-commit` before committing — it runs format, lint, typecheck, and tests via `.mise/tasks/pre-commit`. There is no auto-installed git hook; the worktrunk workflow (`wt step commit`) invokes it.
